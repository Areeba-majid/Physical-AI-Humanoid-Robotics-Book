# AI Textbook Platform Frontend

This is the frontend for the AI-native Physical AI & Humanoid Robotics textbook platform, built with Docusaurus.

## Deployment
This project is configured for deployment on Vercel.

## Installation

```bash
npm install
```

## Local Development

```bash
npm start
```

This command starts a local development server and opens up a browser window. Most changes are reflected live without having to restart the server.

## Build

```bash
npm run build
```

This command generates static content into the `build` directory and can be served using any static contents hosting service.

## Deployment

Using SSH:

```bash
USE_SSH=true npm run deploy
```

Not using SSH:

```bash
GIT_USER=<Your GitHub username> npm run deploy
```

Project Overview:
- Full interactive textbook with multi-chapter navigation.
- Built-in RAG chatbot with citation support.
- Select-text QA (user highlights text to ask questions).
- Urdu translation with proper RTL layout.
- User personalization based on metadata.
- Authentication system (signup/login) and BetterAuth bonus.
- User dashboard showing progress and recommendations.
- Summaries and quizzes (MCQs + short answer) generated by AI.
- Multi-agent architecture with reusable intelligence.
- Clean, responsive, modern frontend + backend.

UI/UX Requirements:
- **Professional typography:** readable, consistent, high contrast.
- **Dark/light mode toggle** with smooth transitions.
- **Color scheme:** Modern, clean, tech-educational feel (e.g., navy blue, teal, white, gray accents). Use a secondary accent color for interactive elements (buttons, highlights).  
- **Responsive design:** Desktop, tablet, mobile. Use proper spacing, margins, and fluid grids.  
- **Interactive elements:** Hover effects on buttons, chapters, links.  
- **RAG Chatbot:** Floating button, expandable overlay, clean chat bubbles, distinguish user vs AI messages, citations in small italic text.  
- **Urdu translation:** RTL layout, text alignment correct, proper font for Urdu (Noto Nastaliq or similar).  

Frontend Stack:
- **Docusaurus** for textbook content.
- **React + TypeScript** for interactive components.
- **Tailwind CSS** for responsive design.
- Proper component modularization for chapters, chatbot, dashboard.

Backend Stack:
- **FastAPI** with proper REST endpoints.
- **MongoDB** for user profiles, progress, translation jobs, and quizzes.
- **Qdrant Cloud** for embeddings and RAG retrieval.
- **Gemini 2.5 Flash** for AI generation (summaries, quiz, chatbot).

Multi-Agent Architecture:
- **Subagents:** Planning agent, code-generation agent, UI agent, file-writing agent.

- Agents should reuse intelligence across features.

Performance Requirements:
- Load in < 2 seconds on broadband.
- Chatbot latency < 2 seconds.
- Support ≥100 concurrent users.
- WCAG AA accessibility compliance.

Edge Cases:

- Urdu translation of code blocks → skip code translation.
- Very long chapters → chunk without breaking sentences.
- Chatbot question outside textbook → respond politely with limitation.
